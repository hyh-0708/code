Week9
This week was our last hands-on Arduino opportunity in class. In this build, our focus shifted from basic circuits to how we could connect and interact with the Arduino in tandem with Processing, specifically, with p5.js code via Web Serial. Although it felt a bit complicated at first, it was really fun and satisfying to see the physical components control the visual changes on the screen
A potentiometer is connected to an Arduino Uno (A0 analog input). The Arduino reads the analog value and maps it from a 10-bit range (0–1023) to a smaller 8-bit range (0–255). The mapped value is sent over the serial port to the computer every 50 milliseconds.
serial.js is used to establish a Web Serial connection in the browser. Incoming serial data (0–255) is mapped to control the size of a circle on screen. The circle is green and positioned at the center; its size responds in real time to the potentiometer input. The background color also changes according to the data received, giving an added layer of feedback. There is also a mouseMoved() function, allowing mouse positions to be sent back to Arduino, showing two-way communication is possible.
This was my first time connecting a physical sensor to the browser and seeing my interaction change something on screen in real time. It was both surprising and satisfying to watch the green circle grow and shrink based on how I turned the knob. The moment the serial data flowed through and the visual changed—everything clicked.
Although I’m still new to physical computing, this class has helped me build confidence. Tools and concepts that once felt intimidating now feel accessible. Even though that confidence is often driven by assignment deadlines, it's real progress nonetheless. I now see how sensors, data, and visuals can be combined into interactive design—and that’s something I’d love to explore more in future creative projects.
